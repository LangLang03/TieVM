# ADR-0001: TieVM v0.x 总体架构

## 状态

Accepted

## 决策

- VM 采用 64 位寄存器解释器模型。
- 字节码采用 `.tbc` 固定长度指令格式，固定小端序列化。
- 模块容器采用 `.tlb`，支持热加载（安全点原子切换）。
- OOP 采用多继承 + 接口，方法解析使用 C3 线性化。
- GC 采用分代并发标记，支持短时 pin + 句柄表。
- FFI 采用显式所有权注解与显式线程 attach/detach。
- AOT 首版只提供 IR/元数据占位接口。

## 影响

- API 稳定性：v0.x 内允许不兼容字节码变更。
- 可扩展性：AOT 后端可通过占位接口接入，不影响解释器主线。
