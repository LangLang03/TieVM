# ADR-0005: 速度优先执行模型（快指令集 + 执行期信任模式）

## 状态
Accepted

## 背景
当前解释器在热循环中的开销主要来自：
- 每条指令分发成本；
- `Value` 通用路径的类型访问与分支；
- 执行期寄存器边界与跳转边界检查。

在大循环场景（如 10^9 次）下，上述开销会被放大。项目当前目标是“速度优先”，并且字节码来源可控。

## 决策
- 扩展高频快指令，减少通用指令组合开销：
  - `add_imm`
  - `sub_imm`
  - `jmp_if_zero`
  - `jmp_if_not_zero`
  - `dec_jnz`
- 解释执行阶段进入“信任模式”：
  - 关闭执行期寄存器索引边界检查；
  - 关闭执行期 `jmp/jmp_if` 目标边界检查；
  - 仍保留加载期（Verifier）合法性校验作为前置保障。
- 性能基准优先使用快循环路径（`dec_jnz`）验证收益。

## 影响
正向影响：
- 热循环路径显著加速，减少解释器每轮额外分支与检查。
- 字节码层可表达更短、更直接的循环与立即数算术序列。

负向影响：
- 若非法/损坏字节码绕过加载期校验，运行期可能触发未定义行为（越界访问或崩溃）。
- 调试期可观测性下降：部分运行期错误不再在边界处给出结构化错误，而可能直接故障。

## 约束与防护
- 所有生产输入必须先通过 Verifier。
- 不再承诺“运行时兜底安全性”，承诺转移为“加载前校验安全性”。
- 对外发布字节码需保证构建链路不可绕过校验步骤。

## 回滚策略
- 若后续需要提高健壮性，可恢复执行期检查，或提供可切换模式：
  - `safe`（执行期检查开启）
  - `fast`（执行期检查关闭，当前策略）
