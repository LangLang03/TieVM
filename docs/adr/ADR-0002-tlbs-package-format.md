# ADR-0002: `.tlbs` 包格式与加载策略

## 状态
Accepted

## 背景
原 `tlb` 仅承载字节码与简单插件名，不足以描述可部署的本地库分发。项目需要统一承载：
- 纯字节码库
- 纯本地库
- 混合库（字节码 + 本地库）

## 决策
- 引入 `.tlbs` 作为标准库与本地扩展分发主格式。
- 同后缀双格式：
  - 目录包：`.tlbs` 目录
  - 压缩包：`.tlbs` 文件（zip store）
- 固定布局：
  - `manifest.toml`（仅包级信息）
  - `modules/*.tbc`
  - `libs/<platform>/<arch>/*`
- FFI 签名与符号映射不放在 `manifest.toml`，全部内嵌在 `.tbc` 扩展段。
- `ModuleLoader` 自动识别目录或 zip；zip 先物化到临时目录后加载。

## 影响
- 提升了可部署性和跨平台打包能力。
- 保持了 `manifest.toml` 简洁，减少包级/模块级元数据重复。
- 带来额外加载成本（zip 物化），但简化了运行时动态库路径处理。
